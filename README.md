# THAI-GO Engine 🇹🇭

Google Apps Script (GAS) をバックエンドとし、Vanilla JS + Tailwind CSS で構築されたタイ語学習のための超高速SPA（Single Page Application）。
モバイル（特にiOS）における極限のUXと、タイ語の複雑な音節・声調ルールを自力で解析する独自のパースエンジンを統合した、現場特化型のパーソナル学習プラットフォームです。

## 🏗 System Architecture (システム構成とデータフロー)

本システムは、GAS環境の制約（ロード時間の長さ、ステートレスな通信）を克服するため、フロントエンド側で強力な状態管理（State Management）を行うアーキテクチャを採用しています。

```text
📦 thai-go-engine
 ┣ 📂 Backend (GAS / Server-side)
 ┃ ┣ 📜 core.gs         # ルーティング、HTMLテンプレート生成、初期データロード
 ┃ ┣ 📜 db_access.gs    # Google SheetsをDBとしたCRUD操作（排他制御付き）
 ┃ ┣ 📜 feat_study.gs   # 学習アルゴリズム（間隔反復）のサーバー側処理
 ┃ ┗ 📜 api_ai.gs       # Gemini API / Cloud TTS API との通信インターフェース
 ┃
 ┣ 📂 Frontend (Vanilla JS / Client-side)
 ┃ ┣ 📜 AppState.html     # アプリ全体の「唯一の真実（Single Source of Truth）」となる状態管理
 ┃ ┣ 📜 Bridge.html       # `google.script.run` をラップした非同期通信レイヤー（Promise化）
 ┃ ┣ 📜 ViewController.html # DOM操作、CSSアニメーション、物理的スクロールロックの司令塔
 ┃ ┣ 📜 VocabListUI.html  # 万能検索窓（Omnibox）とインクリメンタルサーチのロジック
 ┃ ┣ 📜 StudyEngine.html  # 学習UI、キュー管理、相互リンク解決、統計グラフ描画
 ┃ ┣ 📜 AudioSystem.html  # Cloud TTS / Web Speech API のハイブリッド音声制御
 ┃ ┣ 📜 ChatSystem.html   # AI対話UI、動的単語抽出（スナイパー）ロジック
 ┃ ┗ 📜 ThaiData.html     # 【コア技術】タイ文字の音節分解・声調計算パーサー
 ┃
 ┗ 📜 css_main.html     # Tailwind CSS（CDN）とカスタム物理CSSの定義

```

## 🌟 Deep Dive: Core Modules (コアモジュールの詳細仕様)

### 1. ⚙️ 独自実装のタイ語解析エンジン (`ThaiParser` & `ThaiValidator`)

本システムの心臓部。外部APIに依存せず、ブラウザ上でタイ文字の複雑な音節構造と声調ルールをリアルタイムに計算します。

* **音節分解 (Syllable Parsing)**:
前置母音（เ, แ, โ, ใ, ไ）から始まり、末子音で終わるタイ語特有の構造を左からスキャン。二重子音（偽のクラスタ）、Ro Han (รร)、黙字記号（ガーラン: ์）を的確に切り分けます。
* **声調の絶対計算 (Tone Calculation)**:
切り出した音節に対し、文字のクラス（高中低）、母音の長さ（長短）、末子音の性質（平音/促音・生音/死音）、声調記号の有無を判定し、0〜4（平・低・下・高・上）の声調番号を全自動で導き出します。
* **カスタム分割 (Custom Split)**:
機械的なパースが不可能な例外単語（例: トラなどの変則読み）に対し、ユーザーが手動で分割位置を指定できるオーバーライド機能を備えています。

### 2. ⚡️ 万能検索窓「Omnibox」 (`VocabListUI`)

「検索・追加・編集」の境界をなくし、現場で0.1秒で目的の単語にアクセスするためのUI。

* **ディープ・インクリメンタルサーチ**:
タイ語、日本語、発音記号だけでなく、**「解説文 (explanation)」と「例文 (example)」の中身までリアルタイムに全文検索**。ヒットした箇所は前後の文脈を含めてスニペットとして抽出し、ハイライト（`<b class="...">`）表示します。
* **単方向データフロー (Single Source of Truth)**:
ヘッダーの入力欄（Add）とリストの検索窓（Search）で値が競合しないよう、`this.state.rawInput` のみを正とし、DOMをそれに追従させる堅牢な同期ロジックを実装。ペースト操作やスワイプ開閉時でも値がねじれません。

### 3. 🤖 AI統合チャット＆動的単語抽出 (`ChatSystem`)

単なるチャットUIにとどまらず、AIの回答をシステム内のデータとして直接取り込むパイプライン。

* **抽出スナイパーロジック**:
Geminiの返答テキストから、`タイ文字 (発音記号) : 日本語意味` といった特定のフォーマットを正規表現で強欲に抽出し、画面上に「単語カード追加用リスト」として動的にUI生成します。
* **重複検知と自動ルーティング**:
抽出された単語がすでにDB内に存在するかをリアルタイムに走査。未登録なら「Add」ボタン、登録済みなら対象のカードへ直接ジャンプする「登録済 ↗」ボタンへとシームレスに切り替えます。
* **画像解析対応 (Base64 Encoding)**:
ユーザーがアップロードした画像をCanvas経由でBase64にエンコードし、現場の看板やエラー画面の写真をそのままAIのプロンプトとして送信可能です。

### 4. 🧠 忘却曲線ベースの学習エンジン (`StudyEngine`)

Spaced Repetition System (SRS) をベースにしたカード学習システム。

* **3段階の忘却度判定**:
「簡単 (3)」「普通 (2)」「難しい (1)」の評価に基づき、次回の出題日（Interval）を指数関数的に計算。
* **フラッシュバック・ヒートマップ**:
`frappe.Chart` を統合し、過去の学習アクティビティ（Activity）をGitHubの草（コントリビューション）グラフのように可視化。
* **自動コンテキストリンク**:
解説や例文の中に、すでにDBに登録されている別のタイ単語が含まれていた場合、それを自動検知して青文字のクリッカブルリンクに変換。タップでツールチップ表示、ダブルタップで該当カードへジャンプする逆引きシステムを搭載。

### 5. 🎧 ハイブリッド音声システム (`AudioSystem`)

* **シームレスなAPIフォールバック**:
高音質な Google Cloud TTS をメインとしつつ、オフラインや制限時にはブラウザ標準の Web Speech API (`speechSynthesis`) へと自動・手動でフォールバックします。
* **Base64 インメモリキャッシュ**:
Cloud TTSから取得した音声データは、リクエストの都度JSのオブジェクト（`audioCache`）にBase64文字列として保持され、同一セッション内の無駄なAPIコールを完全にゼロにします。

## 🛠 Technical Highlights & Hacks (技術的な見どころ)

このプロジェクトでは、モバイルSafari/WebKitの厳しい仕様や制約を突破するため、いくつかのアグレッシブなハックが組み込まれています。

* **iOS キーボードの物理ハック (Forced Focus)**
iOSでは「ユーザーの直接のタップ操作」以外でのJSからの `focus()`（キーボード起動）がブロックされます。これを突破するため、右下のフローティングボタン（`floating-action-btn`）上に**完全に透明なダミーの `<input>` 要素**を被せ、OSレベルで強制的にキーボードをせり上げつつ、裏側のJSで本命の検索窓へフォーカスをバトンパスする究極の物理ハックを実装しています。
* **Tailwind CSS の限界を超える物理DOM制御**
長文フレーズや複数行の意味を持つ単語の場合、単なるCSSクラスの切り替えではレイアウトが破綻します。そのため `updateText` 関数内にて、文字数や行数をJSで動的にカウントし、`clamp(1.5rem, 6vw, 2.2rem)` のような可変フォントサイズを `style.setProperty('important')` で直接注入し、完璧な中央揃えと視認性を維持しています。
* **GASの非同期カプセル化 (Bridge Pattern)**
GASの `google.script.run` はコールバック形式（`.withSuccessHandler`）でしか記述できず、複雑な処理ではコールバック地獄に陥ります。これを `Bridge` オブジェクト内で全て `Promise` でラップし、フロントエンド全体で `async/await` によるモダンで直感的なデータフローを実現しています。
