# THAI-GO Engine 🇹🇭

Google Apps Script (GAS) をバックエンドとし、Vanilla JS + Tailwind CSS で構築されたタイ語学習のための超高速SPA（Single Page Application）。
モバイル（特にiOS）における極限のUXと、タイ語の複雑な音節・声調ルールを自力で解析する独自のパースエンジンを統合した、現場特化型のパーソナル学習プラットフォームです。

## 🏗 System Architecture (システム構成とデータフロー)

本システムは、GAS環境の制約（ロード時間の長さ、ステートレスな通信）を克服するため、フロントエンド側で強力な状態管理（State Management）を行うアーキテクチャを採用しています。

```text
📦 thai-go-engine
 ┣ 📂 Backend (GAS / Server-side)
 ┃ ┣ 📜 core.gs         # ルーティング、HTMLテンプレート生成、初期データロード
 ┃ ┣ 📜 db_access.gs    # Google SheetsをDBとしたCRUD操作（排他制御付き）
 ┃ ┣ 📜 feat_study.gs   # 学習アルゴリズム（間隔反復）のサーバー側処理
 ┃ ┗ 📜 api_ai.gs       # Gemini API / Cloud TTS API との通信インターフェース
 ┃
 ┣ 📂 Frontend (Vanilla JS / Client-side)
 ┃ ┣ 📜 AppState.html     # アプリ全体の「唯一の真実（Single Source of Truth）」となる状態管理
 ┃ ┣ 📜 Bridge.html       # `google.script.run` をラップした非同期通信レイヤー（Promise化）
 ┃ ┣ 📜 ViewController.html # DOM操作、CSSアニメーション、物理的スクロールロックの司令塔
 ┃ ┣ 📜 VocabListUI.html  # 万能検索窓（Omnibox）とインクリメンタルサーチのロジック
 ┃ ┣ 📜 StudyEngine.html  # 学習UI、キュー管理、相互リンク解決、統計グラフ描画
 ┃ ┣ 📜 AudioSystem.html  # Cloud TTS / Web Speech API のハイブリッド音声制御
 ┃ ┣ 📜 ChatSystem.html   # AI対話UI、動的単語抽出（スナイパー）ロジック
 ┃ ┗ 📜 ThaiData.html     # 【コア技術】タイ文字の音節分解・声調計算パーサー
 ┃
 ┗ 📜 css_main.html     # Tailwind CSS（CDN）とカスタム物理CSSの定義

```

## 🌟 Deep Dive: Core Modules (コアモジュールの詳細仕様)

### 1. 🧠 忘却曲線ベースの3段階学習エンジン (StudyEngine)

* **3段階評価システム**: カードめくり後、「簡単 (3)」「普通 (2)」「難しい (1)」の3ボタンから忘却度を判定し、次回の出題日を自動計算します。
* **ヒートマップ可視化**: 過去の学習実績（Activity）を `frappe.Chart` を用いて視覚的にトラッキング可能です。
* **スマート/クラシックモード**: アルゴリズムに基づく出題（🧠）と、単なるランダム（✨）を即座に切り替えられます。

### 2. ⚡️ 万能検索窓「Omnibox」＆ リアルタイム辞書 (VocabListUI)

* **超広範インクリメンタルサーチ**: タイ語、日本語、発音記号、品詞に加え、「解説文 (explanation)」**や**「例文 (example)」の中身までリアルタイムに検索し、スニペットとしてハイライト表示（`<b class="...">`）します。
* **iOSセキュリティ突破ハック**: 画面右下のフローティングボタンに「透明なダミーInput要素」を被せることで、iOS特有のキーボード起動制限を物理的に突破し、1タップで検索窓へフォーカスを移します。
* **単方向データ同期**: ヘッダーの入力欄（Add）とリストの検索窓（Search）で、ペーストやスワイプ開閉時でも値がねじれない堅牢な同期ロジック（`VocabListUI.update`）を実装しています。

### 3. ⚙️ タイ語特化の独自解析エンジン (ThaiParser & Validator)

システム内部でタイ語の文字列を音節レベルに分解・解析する強力なエンジンを搭載。

* **高精度な音節分解**: `ThaiParser.parseSyllables` により、前置母音、核子音、Ro Han (รร)、末子音を左から舐めるように吸い込んで解析します。
* **声調・生音/死音の自動計算**: `ThaiValidator` が文字のクラス（高・中・低）と母音の長さ、末子音の性質から、タイ語の絶対ルールに従って声調番号（0〜4）を全自動計算します。
* **自己診断とリカバリー**: 裸の子音などを検知した場合、前後の音節と強制吸着を試みる「リカバリー・レイヤー」を持ち、解析エラーの疑いがある場合は画面上に警告（⚠️）を出力します。
* **カスタム分割対応**: どうしても機械的にパースできない例外単語（ทย など）に対しては、ユーザーが手動で分割を指定できる `custom_split` 機能を備えています。

### 4. 🤖 AI統合チャット＆動的カード化 (ChatSystem)

アプリ内で直接AI教師（Gemini）と対話できるシート UI を実装。

* **2つのモード切替**: 「🔁 翻訳・カード化（translate）」と「🤖 先生に質問（ask）」のモードを切り替え、目的に応じたプロンプトを送信します。
* **単語の自動抽出スナイパー**: AIの返答テキストから、正規表現を用いて「タイ語・発音記号・日本語」のペアを自動抽出し、画面上に登録用リストとして動的に生成します。
* **重複チェックとジャンプ**: 抽出された単語がすでにDBに存在するかチェックし、未登録なら「追加」、登録済みなら対象のカードへジャンプするボタン（登録済 ↗）へと切り替えます。
* **画像解析対応**: 画像をBase64にエンコードして送信し、現場の看板やエラー画面の写真をそのままAIのプロンプトとして利用可能です。

### 5. 🔗 網羅的な相互リンクと逆引きシステム (StudyEngine)

* **自動コンテキストリンク**: 例文（`example_th`）や解説（`explanation`）のテキスト内に、既に単語帳に登録されているタイ語が含まれている場合、自動的に青文字のリンク（`context-link`）に変換します。
* **タップでツールチップ**: リンクを1回タップすると独立したツールチップで意味を表示し、2回タップするとその単語のカードへ直接ジャンプします。
* **逆引き表示（References）**: 今見ている単語を「解説」や「例文」に含んでいる「他の単語カード」を抽出し、カード下部にリンクチップとして表示します。

### 6. 🎧 ハイブリッド音声と現場特化の AutoPlay (AudioSystem & AutoPlayEngine)

歩きながらでも「耳だけ」で学習を進められる、シャドーイング特化の無限ループ再生機能。

* **シームレスなAPIフォールバック**: ヘッダーのボタンで、Google Cloud TTS（高音質）とブラウザ標準の Web Speech API を即座に切り替えます。
* **Base64 インメモリキャッシュ**: Cloud TTSで取得した音声データはJSオブジェクトにキャッシュされ、同一セッション内の無駄なAPIコールを完全にゼロにします。
* **スリープ防止とバックグラウンド検知**: 再生中は WakeLock API で画面スリープを物理的に防ぎ、別アプリに切り替わった際は `visibilitychange` で自動一時停止するセーフティを搭載。
* **正確なテキストハイライト追従**: 再生中のタイ語・日本語に合わせて画面テキストをハイライト。同じ単語が複数存在する場合、「テキスト長の差分 (`Math.abs`)」を計算して最適なDOMを狙い撃ちする高度な検索ロジックを実装しています。

### 7. 🪄 AIによるカード再生成と自動メモ追記 (Bridge & ChatSystem)

単語帳のメンテナンス自体をAIに丸投げする高度なインテグレーション。

* **カードの再生成 (AI Re-Generate)**: 「例文がしっくりこない」「解説が足りない」時、`AI Re-Generate` ボタンを押すことで既存IDの解説や例文をAIが丸ごと作り直します。API負荷軽減のため20秒のクールダウンロック（`_startReGenCooldown`）が走ります。
* **AI メモ教師 (Ask AI)**: 学習中の単語について、メモ欄に質問を書き込んで `🪄 Ask AI` ボタンを押すと、AIの回答が特定のデリミタ（区切り文字）と共にメモ欄へ自動追記され、そのままDBへオートセーブされます。

## 🛠 Technical Highlights & Hacks (技術的な見どころ)

このプロジェクトでは、モバイルSafari/WebKitの厳しい仕様や制約を突破するため、いくつかのアグレッシブなハックが組み込まれています。

* **iOS キーボードの物理ハック (Forced Focus)**
iOSでは「ユーザーの直接のタップ操作」以外でのJSからの `focus()`（キーボード起動）がブロックされます。これを突破するため、右下のフローティングボタン（`floating-action-btn`）上に**完全に透明なダミーの `<input>` 要素**を被せ、OSレベルで強制的にキーボードをせり上げつつ、裏側のJSで本命の検索窓へフォーカスをバトンパスする究極の物理ハックを実装しています。
* **Tailwind CSS の限界を超える物理DOM制御**
長文フレーズや複数行の意味を持つ単語の場合、単なるCSSクラスの切り替えではレイアウトが破綻します。そのため `updateText` 関数内にて、文字数や行数をJSで動的にカウントし、`clamp(1.5rem, 6vw, 2.2rem)` のような可変フォントサイズを `style.setProperty('important')` で直接注入し、完璧な中央揃えと視認性を維持しています。
* **GASの非同期カプセル化 (Bridge Pattern)**
GASの `google.script.run` はコールバック形式（`.withSuccessHandler`）でしか記述できず、複雑な処理ではコールバック地獄に陥ります。これを `Bridge` オブジェクト内で全て `Promise` でラップし、フロントエンド全体で `async/await` によるモダンで直感的なデータフローを実現しています。
